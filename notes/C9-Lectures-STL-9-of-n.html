C9 Lectures - Advanced STL Part 9 of n

published on September 29, 2013

  • lvalues are expressions that have a persistent location in memory. You can take their address (&).

  • rvalues are temporaries without a persistent location in memory. (STL: They (usually) die at the semicolon.)

  • move constructors and move assignment operators

    class RI // remote integer
{
  int* _p;
public:
  explicit RI(const int n) {
    _p = new int(n);
  }
  ~RI() {
    delete _p;
  }

  // copy constructor
  RI(const RI& other) {
    _p = other._p ? new int(*other._p) : nullptr;
  }

  // move constructor
  RI(RI&& other) /* note the && (ref-ref) */ {
    _p = other._p;
    other._p = nullptr;
  }

  RI& swap(RI& other) {
    swap(this->_p, other._p);
    return *this;
  }

  // copy assignment operator
  RI& operator=(const RI& other) {
    RI(other).swap(*this);
    return *this;
  }

  // move assignement operator
  RI& operator=(RI&& other) {
    // other has a name, so (as an expression) is a lvalue. (see next point).
    RI(std::move(other)).swap(*this);
    return *this;
  }
};

  • The differentiation of lvalue and rvalue applies to expressions, not objects.

  • std::move is the logical equivalent of a cast. It casts an lvalue reference to an rvalue reference.

  • universal assignment operator:

    class A
{
  A& operator=(A other) {
    A.swap(*this);
    return *this;
  }
}
